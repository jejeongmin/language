C++ 는 객체 생성을 과정을 다음과 같이 정의하고 있다.

1. 베이스 클래스라면 디폴트 생성자를 실행한다. 단 생성자 이니셜라이저가 있다면 디폴트 생성자 대신 생성자 이니셜라이저를 호출한다.
   생성자 이니셜라이저 설명 -> https://koey.tistory.com/62
2. static 으로 선언하지 않은 데이터 멤버를 코드에 나타난 순서대로 생성한다.
3. 현재 클래스 생성자의 본문을 실행한다.

객체를 삭제할 때는 반대의 순서로 진행된다.

1. 현재 클래스의 소멸자를 호출한다.
2. 현재 클래스의 데이터 멤버를 생성할 때와 반대의 순서로 삭제한다. stack 구조니깐..
3. 부모 클래스가 있다면 부모의 소멸자를 호출한다.

static 변수는 heap에 stack 보다 먼저  올라온다.
