함수 인자건, 복사 생성자 건 인자를 전달할 때 레퍼런스로 전달해야, 큰 객체를 전달할 경우 복사로 인한 성능 문제를 미연에 방지할 수 있다.
그리고 복사 원본이 되는 객체의 변경을 막기 위해 const 로 전달해야 한다.

int, double 과 같은 기본 타입은 const 레퍼런스로 전달한다고 해서 나아지는 건 없다. 그냥 값으로 전달해도 된다.

함수 인자의 경우는 큰 객체를 전달할 때, const 레퍼런스로 전달하는 게 선언상으로 눈에 보이는데
객체의 복사 생성자의 경우는 명시적으로 복사 생성자를 삭제함으로 미연에 성능 문제를 방지하도록 할 수 있다.

ex)

SpreadsheetCell(const SpreadsheetCell& src) = delete;
